# Soap data catalog

## Назначение skill
Вести проверяемый каталог мыла, ПАВ, сырья и автохимии: сохранять исходные сведения, отделять факты от гипотез и не завершать работу без проверок.

## Условия автоматического запуска
Вызвать `$soap-data-catalog` до изменений, если нужно добавить, исправить, импортировать, классифицировать, объединить, удалить или проверить карточку продукта; перенести данные из этикетки, накладной, SDS/TDS или паспорта качества; изменить схему, шаблон или валидатор. Синонимы вроде «занеси товар», «сохрани компонент», «разбери этикетку» тоже активируют skill.

## Когда skill не использовать
Не применять для несвязанных программных задач, общего объяснения химии без изменения файлов, подбора поставщика или рецептуры без сохранения в каталог, а также для данных о сотрудниках, клиентах и камерах.

## Необходимые входные данные
Минимум: точный исходный текст и тип источника. Желательно: фото этикетки, производитель, артикул, партия, масса, концентрация, дата и ссылка на документ. При нехватке данных создать `needs-verification` и перечислить, что подтвердить.

## Какие файлы проекта сначала изучать
1. `AGENTS.md`
2. `README.md`
3. `schemas/product.schema.json`
4. `assets/product-template.json`
5. релевантные `data/products/*.json`
6. `scripts/validate_catalog.py` и тесты
7. `references/data-model.md`
8. `references/lessons-learned.md`

## Пошаговый рабочий процесс
1. Активировать skill до правок.
2. Сохранить исходную строку без молчаливого исправления.
3. Проверить дубли по id, названию, производителю и артикулу.
4. Разнести сведения в `confirmed_facts` и `hypotheses`.
5. Выбрать `needs-verification`, `verified` или `deprecated`.
6. Создать карточку через `scripts/add_product.py` либо точечно изменить существующую.
7. Обновить `updated_at`, не подменяя дату источника.
8. Запустить обязательные проверки и просмотреть diff.
9. Оценить, появился ли подтверждённый переиспользуемый урок.
10. Подготовить итоговый отчёт.

## Команды и инструменты
```bash
python scripts/validate_catalog.py
python scripts/check_skill.py
python -m unittest discover -s tests -v
python scripts/add_product.py --id <id> --name "<name>" --source-type <type> --source-text "<text>"
```
Для изображений извлекать только читаемые данные. Не додумывать символы.

## Обработка ошибок и откат
При невалидном JSON исправить конкретную ошибку и повторить тесты. Конфликтующие источники сохранять вместе и понижать статус до `needs-verification`. Дубли не удалять автоматически. Не использовать `reset --hard`; откатывать только собственные изменения после просмотра diff.

## Обязательные проверки
1. Все JSON читаются и имеют обязательные поля.
2. `id` в kebab-case и совпадает с именем файла.
3. Статус и категория допустимы.
4. Есть хотя бы один источник.
5. `verified` имеет подтверждённые факты; `needs-verification` — следующие шаги.
6. Гипотезы не записаны как факты.
7. Нет секретов и лишних персональных данных.
8. Три команды проверки завершаются успешно.
9. Изменения не противоречат `AGENTS.md` и не затрагивают посторонние файлы.

## Критерии готовности
Карточка создана или обновлена; исходник сохранён; факты и гипотезы разделены; неопределённость видна; источники и дальнейшая проверка указаны; все тесты успешны; итоговый отчёт готов.

## Формат итогового отчёта пользователю
Сообщить изменённые записи и файлы, подтверждённые факты и источники, оставшиеся гипотезы, результаты трёх проверок, недостающие данные и факт обновления skill.

## Правила накопления подтверждённого опыта
Автоматически добавлять небольшой урок только если он исправляет подтверждённую ошибку, проверен тестом или пользователем, пригоден повторно, не содержит секретов и не противоречит проекту. Перед изменением цели, архитектуры, обязательных команд, безопасности или формата результата запросить разрешение. Не удалять рабочее правило из-за одной задачи.

## Запрещённые действия
Не выдавать догадку за факт; не исправлять исходник молча; не ставить `verified` без источника; не объединять только по похожему названию; не сохранять пароли и токены; не удалять историю уроков; не менять серьёзно skill без разрешения; не завершать работу при проваленных проверках.
